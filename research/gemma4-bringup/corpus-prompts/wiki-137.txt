--- title: Discriminated-union tool results type: concept tags: [llm-tooling, agent-tools, api-design, error-handling, structured-output] sources: [] updated: 2026-06-29 --- # Discriminated-union tool results A *discriminated-union tool result* is a design pattern for the value an LLM-facing tool returns to an agent: instead of a free-form string (or an exception that escapes the tool), the tool always returns a structured object whose first field is a discriminator tag — conventionally `kind` — and whose remaining fields are determined by the value of that tag. A `read` tool might return `kind: "text"`, `kind: "directory"`, `kind: "attachment"`, or `kind: "error"`, with each branch carrying a different payload. The bet behind the pattern is that a probabilistic model parses an explicit tag like `kind: "not_found"` far more reliably than it parses an error marker embedded in prose, so the discriminator becomes the single thing the model branches on when deciding its next move.
