--- title: progressive-tool-loading type: concept tags: [mcp, agents, context-engineering, tool-use, token-efficiency, skills] sources: [] updated: 2026-06-29 --- # progressive-tool-loading Progressive tool loading (also called progressive disclosure) is a context-engineering pattern for agents that connect to many tools: instead of serializing every tool's full schema into the system prompt at session start, the agent sees only a lightweight index — typically `(server, tool, one-line description)` tuples at roughly 10-30 tokens each — and the verbose JSON schemas, parameter descriptions, and example payloads load on demand only when a tool is actually invoked.[^1] The technique is not a protocol change. It works on the current [[model-context-protocol]] spec because MCP already separates discovery (`tools/list`) from invocation; what changed in 2026 is the assumption that a client must flatten the discovery result into one big preloaded catalog.[^1] The motivation is arithmetic: by Q1 2026 a typical production agent connected to five to twenty MCP servers, each exposing five to fifty tools, each schema rarely under 300 tokens and often over 1,000 — so ten servers at twenty tools and 500 tokens each is 100,000
