--- title: Write–Manage–Read Loop type: concept tags: [agent-memory, taxonomy, pomdp, policy-learned-management, survey] sources: [] updated: 2026-06-29 --- # Write–Manage–Read Loop The **write–manage–read loop** is the organizing abstraction for agent memory introduced by a March 2026 survey, *Memory for Autonomous LLM Agents* (Pengfei Du et al., arXiv 2603.07670), which covers work from 2022 through early 2026.[^1] Rather than treating memory as a static store the agent reads from, the survey formalizes it as a continuous cycle embedded inside the agent's POMDP dynamics. At each step the policy chooses an action by consulting a **read** operation R over current memory — `aₜ = πθ(xₜ, R(Mₜ, xₜ), gₜ)` — and then memory is updated by a function U — `Mₜ₊₁ = U(Mₜ, xₜ, aₜ, oₜ, rₜ)` — where U **summarizes, deduplicates, scores, and deletes**.[^2] The three verbs name the three decisions a memory system must make: what to **write** (commit from the current step), how to **manage** (consolidate, prune, resolve contradictions over time), and what to **read** (retrieve for the next action).
