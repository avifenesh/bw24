--- title: Valkey over RDMA type: concept tags: [valkey, rdma, networking, transport, performance] sources: [] updated: 2026-06-29 --- # Valkey over RDMA Valkey over RDMA is an experimental transport for [[valkey]] that lets clients and servers exchange data using Remote Direct Memory Access instead of the standard TCP socket path. RDMA moves bytes directly between the memory of two machines over the network adapter, bypassing much of the kernel networking stack and the per-byte CPU copies that TCP incurs. The feature was introduced in Valkey 8.0 (release candidate announced 2024-08-02) via PR #477, contributed by pizhenwei, and the project reports up to a **275% increase in throughput** from the direct-memory-access data path between clients and Valkey servers.[^1] The Valkey 8.0 announcement explicitly flags it as experimental, noting the feature "might change or be removed in future versions."[^1] This page exists mainly as the contrast point for the [[udp-diskless-bgsave]] research.
