--- title: agnix type: entity status: active tags: [agent-tooling, linter, agent-sh] sources: [codex-mem-agent-tooling, eigen-mem-agnix] updated: 2026-07-06 --- # agnix agnix is the owner's **agent-config linter**: it validates agent configuration files — skills (SKILL.md), CLAUDE.md / AGENTS.md memory, hooks, MCP configs, agent definitions, plugins, and editor rule files (`.cursorrules`, `.clinerules`, `GEMINI.md`) — against a large, growing rule set (on the order of ~405 rules at the start of this release line, recorded climbing through 426 and counting), with auto-fix. It is a member of the [[agent-sh]] org/monorepo and one of that org's two native **Rust** products (the other being the agent-analyzer behind `/repo-intel`). The Cargo workspace splits into `agnix-core` (the validation engine; the skill validator lives in `crates/agnix-core/src/rules/skill/mod.rs` with its schema helper in `crates/agnix-core/src/schemas/skill.rs` — the two drift unless changed together), `agnix-rules` (generated from the knowledge-base rule JSON), `agnix-cli` (binary `agnix`, published to crates.io as `agnix-cli`), `agnix-lsp` (a tower-lsp server for real-time IDE validation), `agnix-mcp` (an MCP server, verified via the MCP `initialize` handshake rather than `--version`), and `agnix-wasm` (a browser playground).
