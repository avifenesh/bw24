--- title: presigned-url-upload type: concept tags: [storage, s3, r2, uploads, security, sigv4] sources: [] updated: 2026-06-29 --- # presigned-url-upload A presigned URL is a temporary URL that authorizes a single S3 API operation on a single object without exposing the storage account's long-lived credentials. The signature is embedded in the URL's query string, so anyone holding the URL can perform exactly the signed operation (`GET`, `HEAD`, `PUT`, or `DELETE`) on exactly the signed object until the URL expires — while the bucket itself stays private.[^1][^2] For uploads specifically, the pattern is: the app server signs a short-lived `PUT` URL with its credentials, hands that URL to the browser, and the browser uploads the file bytes **directly** to the object store. The server orchestrates but never proxies the payload, which is the whole point — it keeps large media off the app server's request path. In [[nutrical-rev]] this is exactly the documented motivation: `src/lib/r2.ts`'s `presignPut` issues a default-600-second (10-minute) PUT URL so trainee lift videos upload straight to Cloudflare R2, avoiding Railway function timeouts and the CPU cost of streaming large
