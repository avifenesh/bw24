--- title: Structured Tool Output (MCP structuredContent) type: concept tags: [agentic, tool-calling, protocol, json-schema, mcp, constrained-decoding] sources: [] updated: 2026-06-30 --- # Structured Tool Output (MCP structuredContent) Structured tool output is the [[model-context-protocol]] (MCP) feature, introduced in the **2025-06-18** spec revision, that lets a tool return a machine-readable JSON object instead of only a plain-text blob.[^1] In the original protocol a `tools/call` result carried a `content` array of unstructured items (text, image, audio, resource links); a tool that computed structured data had to serialize it into a text string and hope the model parsed it back correctly. The 2025-06-18 revision added a parallel `structuredContent` field on the tool result — a first-class JSON object that the client and model can consume directly.[^1] For backward compatibility a tool returning structured content **SHOULD** also place the serialized JSON inside a `TextContent` block, so older clients that only read `content` still see the data.[^1] The companion half of the feature is the optional **`outputSchema`** on the tool definition (alongside the existing `inputSchema`).[^1] When a tool declares an output schema, servers **MUST** return `structuredContent`
