--- title: Apache Iggy type: entity status: active tags: [rust, message-streaming, io_uring, thread-per-core, compio, apache-incubator] sources: [] updated: 2026-06-29 --- # Apache Iggy Apache Iggy (incubating) is a persistent message streaming platform written in Rust, currently in the Apache Software Foundation incubator.[^1][^2] It is a from-the-ground-up append-only log — explicitly *not* a layer on top of Kafka or an SQL database — built on low-level I/O with a thread-per-core, shared-nothing architecture, `io_uring`, and the `compio` async runtime.[^1] It supports four transport protocols over a custom binary wire specification — QUIC, TCP, WebSocket, and HTTP (a regular REST API) — and claims throughput of millions of messages per second at predictable low latency, helped by Rust's lack of garbage collection.[^1][^2] The name abbreviates "Italian Greyhound," a small, fast dog.[^1] Beyond the core log it ships a broad feature set: multiple streams/topics/partitions, consumer groups with cooperative partition rebalancing, server-side consumer offset storage, polling by offset or timestamp, message expiry with retention policies, server-side deduplication, multi-tenancy via streams grouping topics, optional AES-256-GCM encryption, TLS on all transports, and user auth with Personal Access
