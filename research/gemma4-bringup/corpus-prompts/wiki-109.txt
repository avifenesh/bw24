--- title: Kata Containers type: entity tags: [sandboxing, microvm, kubernetes, oci, kvm, container-runtime] sources: [] updated: 2026-06-29 --- # Kata Containers Kata Containers is an open-source project that runs containers inside lightweight virtual machines, so each workload "feels and performs like a container but provides the isolation and security advantages of a VM."[^1] It launched in December 2017 by merging two predecessor codebases — Intel's **Clear Containers** and Hyper.sh's **runV** — and is hosted by the Open Infrastructure Foundation (the OpenStack Foundation's successor); it is licensed Apache 2.0 with the code at `github.com/kata-containers`.[^1] Crucially, it is not part of OpenStack itself: it is managed by the same foundation but runs equally on Azure, Google Cloud, or bare KVM.[^1] The design goal is to be a drop-in, OCI-compliant runtime so existing images need no rebuild. Kata is **OCI compliant** and speaks the Kubernetes Container Runtime Interface (CRI), so it slots under Docker/containerd or CRI-O and is selected per-pod via a Kubernetes `RuntimeClass` — the standard production wiring described on [[microvm-sandboxing]], where the VM monitor bridges to CRI through that RuntimeClass.[^1] Each
