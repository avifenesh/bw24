--- title: Fine NIF Wrapper type: concept tags: [fine, elixir, nif, cpp, erlang, exla, dashbit] sources: [exla-cpu-only-build] updated: 2026-06-29 --- # Fine NIF Wrapper Fine is a C++ library from Dashbit that makes Erlang/Elixir NIFs (Native Implemented Functions) more ergonomic to write. Erlang ships a raw C API (`erl_nif`) for native functions; Fine is not a replacement but a complementary layer on top of it that removes the boilerplate and ad-hoc helper functions that NIF-heavy projects otherwise copy from repo to repo. Its core features are: automatic encoding/decoding of NIF arguments and return values inferred straight from C++ function signatures, smart pointers (`fine::ResourcePtr<T>`, created via `fine::make_resource<T>`) for safe reference-counted management of resource objects, registration of NIFs and resource types through simple annotation macros (`FINE_NIF`, `FINE_RESOURCE`, `FINE_INIT`), compile-time-metadata-driven encoding of Elixir structs, propagation of C++ exceptions as Elixir exceptions (including custom ones via `fine::raise`), creation of all static atoms at load time, and STL-compatible Erlang-backed mutex/rwlock primitives.[^1] It requires C++17 and works with GCC, Clang, and MSVC; it is licensed Apache 2.0 (copyright 2025 Dashbit) and draws on earlier ideas from
