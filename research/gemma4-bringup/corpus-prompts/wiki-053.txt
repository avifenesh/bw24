--- title: Apache Cassandra type: entity status: active tags: [distributed-systems, database, nosql, consensus, dynamo, masterless] sources: [] updated: 2026-06-29 --- # Apache Cassandra Apache Cassandra is a free, open-source, distributed wide-column NoSQL database built for high write throughput and horizontal scale across many commodity servers, including across multiple data centers.[^1] Avinash Lakshman (a co-author of Amazon's Dynamo) and Prashant Malik wrote it at Facebook to power inbox search; Facebook open-sourced it on Google Code in July 2008, it entered the Apache Incubator in March 2009, and graduated to a top-level Apache project on 17 February 2010.[^1] Architecturally it fuses two lineages: Amazon **Dynamo**'s distribution model (a masterless peer-to-peer ring, a gossip protocol for cluster state, consistent hashing, tunable consistency) with Google **Bigtable**'s storage model (an LSM-tree write path of commit log → memtable → immutable SSTables, with compaction).[^1][^2] It is the textbook **AP** system under CAP: every node is identical so there is no single point of failure, writes stay available during partitions, and consistency is tunable per-query (e.g. quorum reads/writes) rather than strict by default.
