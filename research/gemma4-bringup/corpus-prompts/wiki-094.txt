--- title: The End-to-End Principle type: concept tags: [networking, systems-design, architecture, layering, tcp-ip, reliability] sources: [] updated: 2026-06-29 --- # The End-to-End Principle The end-to-end principle is a design rule for distributed systems, stated formally by Jerome Saltzer, David Reed, and David Clark in their 1984 paper *End-to-End Arguments in System Design* (ACM Transactions on Computer Systems 2(4), 277–288).[^1] It guides where a given function should live among the layers of a layered system. The core claim: a function such as reliable delivery, ordering, or integrity checking can be **completely and correctly implemented only with the knowledge and help of the application sitting at the endpoints** of the communication. A lower layer — the network, the platform — has only partial information and so cannot guarantee the function on its own; any version it implements is necessarily incomplete.[^1] The classic worked example is reliable file transfer from machine A to machine B: the platform layer can checksum and retransmit individual chunks, but only the application can checksum the *assembled file* and decide whether the whole transfer truly succeeded — including
