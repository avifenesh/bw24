--- title: Cloudflare Pages — static-site + JAMstack hosting type: entity tags: [cloudflare, hosting, static-site, jamstack, deploy, edge, frontend] sources: [] updated: 2026-06-29 --- # Cloudflare Pages — static-site + JAMstack hosting Cloudflare Pages is Cloudflare's managed hosting product for static sites and JAMstack/SPA front-ends. You point it at a Git repository (or upload directly), give it a build command and an output directory, and it builds the site and serves the resulting assets from Cloudflare's global edge network. Dynamic logic is added through Pages Functions, which run on the same Workers runtime and are billed at the same rate as Workers. In this wiki it shows up as the recommended deploy target for [[matzai-logistics]], a Vite-built React SPA: build `npm run build`, publish the `dist/` directory, and add a `_redirects` SPA catch-all (`/* /index.html 200`) so client-side routing works.
