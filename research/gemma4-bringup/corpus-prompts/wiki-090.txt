--- title: FlashAttention type: concept tags: [attention, flashattention, io-aware, cuda, kernel, gpu, hardware-squeeze] sources: [] updated: 2026-06-29 --- # FlashAttention FlashAttention is an exact, IO-aware algorithm for computing softmax attention as a single fused GPU kernel. Standard attention materializes the full `L × L` score matrix `S = QK^T` in high-bandwidth memory (HBM), applies softmax, then multiplies by `V` — three passes that read and write the quadratic-size intermediate. On modern GPUs attention is memory-bound, not compute-bound, so those HBM round-trips dominate the runtime. FlashAttention never writes the full matrix: it tiles `Q`, `K`, and `V` into blocks that fit in on-chip SRAM, and uses **online softmax** to combine each block into a running output while carrying only the running max and normalization factor.[^1] This keeps the math exact (it is not an approximation like sparse or [[linear-attention]]) while cutting HBM traffic from `O(L²)` to `O(L²/M)` for SRAM size `M`, turning a memory-bound kernel into a compute-bound one and yielding large wall-clock speedups and linear memory in sequence length.
