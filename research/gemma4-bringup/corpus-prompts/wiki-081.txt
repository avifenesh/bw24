--- title: gn-kernels type: entity tags: [cuda, cutedsl, triton, sm120, sm100, gemm, attention, nvfp4, tensor-core, hardware-squeeze] sources: [bw24-inference-engine] updated: 2026-06-29 --- # gn-kernels `gau-nernst/gn-kernels` is a personal GPU-kernel repository by Thien Tran ("gau-nernst") that collects matmul (GEMM) and attention kernels across three backends — Triton, raw CUDA (via NVRTC), and NVIDIA's Python **CuTe DSL** — and packages them so his other projects can share the code.[^1] It installs as a Python package against `torch>=2.10` (CUDA build must match the local `nvcc`).[^1] The repo is notable because it ships a **separate CuTe DSL matmul kernel per Blackwell target**: an SM100 (datacenter B200) variant built on TMA + `tcgen05`, and a distinct **SM120 (consumer RTX 50-series) variant built on TMA + warp-level `mma.sync`** — exactly the architectural split that [[blackwell-sm120]] forces, where SM120 has no TMEM and cannot issue `tcgen05` (see [[tcgen05]]). The SM120 matmul covers BF16, INT8, FP8, MXFP8, and NVFP4 input dtypes; the older SM80/SM89 matmul uses `cp.async` + `mma.sync` for BF16/INT8/FP8.[^1] Its attention kernels (Triton and CUDA) do GQA + causal with optional INT8/FP8 QK MMA and optional FP16
