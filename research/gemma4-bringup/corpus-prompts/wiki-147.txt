--- title: Tool Poisoning & Line Jumping (MCP security) type: concept tags: [mcp, security, prompt-injection, agents] sources: [research-2026-07-02-aiml-mcp-tool-context-bloat] updated: 2026-07-03 --- # Tool Poisoning & Line Jumping (MCP security) The foundational [[model-context-protocol]] vulnerability: **tool descriptions are injected into model context the moment a client connects, before any tool is called** — a prompt-injection vector that bypasses MCP's "explicit consent before tool call" principle entirely. Disclosed independently as "line jumping" (Trail of Bits) and "tool poisoning" (Invariant Labs), April 2025. It breaks two MCP promises at once: invocation controls (the injection is pre-invocation) and connection isolation (a malicious server can instruct the model to relay between nominally isolated servers). Variants: - **Rug pull** — change a tool description *after* the user approved it. - **Tool shadowing** — a malicious server's description hijacks a *trusted* server's tool; invisible in the invocation log. - The kicker is **UI asymmetry**: the model sees the full description while the user sees a summary, so human approval provides no real protection. Quantified at scale by **MCPTox** (45 live servers, 353 tools, 20 agents): o1-mini attack success
