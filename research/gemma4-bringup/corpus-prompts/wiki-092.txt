--- title: Responses API (OpenAI) type: entity tags: [agents, tool-calling, openai, api, serving, statefulness] sources: [] updated: 2026-06-29 --- # Responses API (OpenAI) The **Responses API** is OpenAI's agent-oriented API primitive, launched in March 2025 as an evolution of the Chat Completions API and positioned as the recommended interface for all new projects.[^1][^2] Where Chat Completions returns an array of `choices`, each wrapping a `message`, Responses returns a typed `output` array of **Items** — a union of distinct types such as `message`, `reasoning`, `function_call`, and `function_call_output` — so each unit of model context is a separate, well-typed object rather than concerns glued into one message.[^2] The SDK exposes a `client.responses.create(...)` call and an `output_text` helper for the common case; the parallel-generation `n` parameter from Chat Completions is removed, leaving one generation per request. OpenAI reports the API has processed trillions of tokens since launch, and cites internal evals showing a 3% SWE-bench improvement and 40–80% better cache utilization (hence lower cost) versus Chat Completions on reasoning models like GPT-5.[^1][^2] The defining property is that the API is **agentic by default**:
