--- title: Zero-Centered RMSNorm type: concept tags: [normalization, rmsnorm, training-stability, quantization, gemma, qwen] sources: [] updated: 2026-06-29 --- # Zero-Centered RMSNorm **Zero-centered RMSNorm** is a re-parameterization of the learnable scale (gamma) in RMSNorm so the stored parameter is centered on zero instead of one. Standard RMSNorm scales the normalized activation by a per-channel weight gamma that is initialized to 1.0, because 1.0 is the identity (no scaling). Zero-centered RMSNorm instead learns a raw parameter omega initialized to 0 and computes the effective scale as `gamma = 1 + omega`, so the forward formula becomes `y = x / RMS(x) * (1 + omega)`.[^1][^2] The identity behavior is preserved at init (omega = 0 gives gamma = 1), but the value that actually lives in memory and gets optimized now sits at zero. NVIDIA's Transformer Engine exposes exactly this as the `zero_centered_gamma` flag on its `RMSNorm`/`LayerNorm` layers ("gamma parameter is initialized to 0 and the formula changes to `(1 + gamma)`"), and HuggingFace's Gemma implementation does the same in code, multiplying by `(1.0 + self.weight)` with the weight initialized to zeros.[^2][^3]
