--- title: io_uring — Linux Shared-Ring Asynchronous I/O type: concept tags: [linux, async-io, kernel, networking, performance, syscall] sources: [research-2026-07-10-systems-completion-async-io] updated: 2026-07-11 --- # io_uring — Linux Shared-Ring Asynchronous I/O io_uring is a Linux kernel asynchronous I/O interface introduced in kernel 5.1 (2019) by Jens Axboe to replace the older, partial, often-blocking `aio` interface. Its core mechanism is a pair of ring buffers shared between userspace and the kernel via `mmap`: a **submission queue (SQ)**, into which the application writes submission queue entries (SQEs) describing I/O requests and advances a tail pointer, and a **completion queue (CQ)**, into which the kernel posts exactly one completion queue event (CQE) per submitted request.[^1][^2] Because the rings live in memory mapped into both address spaces, batches of operations can be submitted and reaped without a system call per operation — and in polled / `SQPOLL` modes a dedicated kernel thread dequeues submissions so that I/O-heavy workloads can run with effectively zero syscalls on the hot path.[^1][^2] The interface is general: not just block reads/writes but sockets (`accept`, `recv`, `send`, `connect`), file opens, timeouts, and
