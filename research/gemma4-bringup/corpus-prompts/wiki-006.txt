--- title: Unified Hugging Face Checkpoint type: concept tags: [quantization, checkpoint-format, deployment, nvidia, modelopt] sources: [] updated: 2026-06-29 --- The **Unified Hugging Face Checkpoint** is the on-disk export format produced by [[tensorrt-model-optimizer|NVIDIA Model Optimizer]] (`nvidia-modelopt`) for quantized and optimized models, designed so that a single artifact can be loaded by several inference frameworks without per-framework conversion. The export keeps the layer structure and tensor names aligned with the original Hugging Face checkpoint, so a quantized model looks like a normal HF model with extra quantization metadata bolted on.[^1] A checkpoint consists of three parts: a group of `safetensors` files holding quantized weights and their scaling factors; a `hf_quant_config.json` file recording the quantization configuration (per-layer `quant_algo`, `kv_cache_dtype`, and similar fields); and the usual HF JSON files for model structure, tokenizer, and metadata.[^1] Models are produced through `modelopt.torch.export.export_hf_checkpoint(model, export_dir)` after PTQ or QAT.[^1] The format replaces the older legacy TensorRT-LLM checkpoint format, which worked only with the TensorRT-LLM C++ backend.[^1] By contrast the unified format targets a deployment matrix spanning TensorRT-LLM (v0.17.0+), vLLM (v0.10.1+), and [[sglang]] (v0.4.10+), letting the same files run
