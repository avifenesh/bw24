--- title: Self-Hosting Compiler type: concept tags: [compilers, bootstrapping, languages, toolchains, trusting-trust] sources: [] updated: 2026-06-29 --- # Self-Hosting Compiler A **self-hosting compiler** is a compiler for a language that is itself written in that same language and can compile its own source code.[^1] More broadly, software is "self-hosted" when the toolchain that produces new versions of it runs on, and is built by, that same software — the classic example being an operating system whose build tools run on the OS itself.[^1] Self-hosting is a strong form of dogfooding: making a compiler compile itself is a non-trivial stress test of the language, it lets all compiler development proceed in the high-level language being compiled (so contributors only need to know that one language), and it doubles as a consistency check, since a correct compiler should be able to reproduce its own object code byte-for-byte.[^1] The first self-hosting compiler (excluding assemblers, which were the first tools to bootstrap themselves) was a Lisp compiler written in Lisp by Tim Hart and Mike Levin at MIT in 1962, tested inside an existing
