--- title: Distributed Shared Memory (DSMEM) type: concept tags: [cuda, hopper, blackwell, thread-block-cluster, shared-memory, smem, kernel] sources: [] updated: 2026-06-29 --- # Distributed Shared Memory (DSMEM) Distributed shared memory is the NVIDIA CUDA feature, introduced with the Hopper (SM90) architecture, that lets the thread blocks of a *thread block cluster* directly read, write, and perform atomics on each other's on-chip shared memory (SMEM).[^1] Before Hopper, a block's SMEM was strictly private — the only way for two blocks to exchange data was a round trip through global memory (and L2). Hopper groups co-scheduled blocks into a cluster whose SMEM windows are mapped into a single distributed address space, so a block can issue a load or store targeting an address in a *sibling* block's SMEM on a different streaming multiprocessor (SM).[^1] The maximum portable cluster size is 8 blocks; an H100 can be pushed to a cluster size of 16 via `cudaFuncAttributeNonPortableClusterSizeAllowed`, potentially at the cost of fewer concurrently active blocks.[^1] Because DSMEM traffic rides a dedicated SM-to-SM network instead of going out to DRAM, it can be used simultaneously
