As vocabulary sizes grow, categorical sampling has become a real LLM-serving bottleneck, and the standard PyTorch path — `sort` (descending) → `softmax` → `cumsum` → mask → re-scatter — is dominated by the full-vocabulary sort, which is expensive and launches several kernels.[^1] The sorting-free approach reframes selection as **inverse-transform sampling over the unsorted probability prefix-sum (CDF)**: draw `u ~ Unif(0,1)`, compute the running prefix sum, and return the first token `k` where `F_{k-1} ≤ u < F_k`. On GPU this is done one threadblock per distribution using NVIDIA CUB/CCCL block primitives (`BlockReduce`, `BlockScan`, `AdjacentDifference`), splitting vocabularies larger than `BLOCK_SIZE` into sequential blocks with early-stopping so a draw rarely scans the whole vocabulary.[^1] This is the design behind [[flashinfer]]'s sampling operators (first shipped in v0.0.5) and the same machinery is reused for
