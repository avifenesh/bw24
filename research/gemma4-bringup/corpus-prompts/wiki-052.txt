--- title: React Server Components (RSC) type: concept tags: [web, react, nextjs, rendering, frontend] sources: [] updated: 2026-06-29 --- # React Server Components (RSC) React Server Components are a type of React component that renders ahead of time, before bundling, in an environment separate from both the client app and the traditional SSR server — the "server" in the name.[^1] Stabilized as an architecture and set of APIs in React 19, RSC lets components run on the server and never ship their code to the browser, making them well suited to fetching data close to the source (databases, APIs), using secrets like API keys without exposing them, and reducing the JavaScript bundle sent to the client.[^2][^3] Components that need interactivity, state, lifecycle effects (`useEffect`), or browser-only APIs (`localStorage`, `window`) instead opt into the client with a `"use client"` directive at the top of the file, which declares a boundary between the server and client module graphs; everything imported below that boundary is bundled for the browser.[^3] Rather than emitting only HTML, a server renders the Server Component tree into a
