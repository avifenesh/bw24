--- title: "Wayland" type: concept tags: [linux, display-server, wayland, compositor, x11, computer-use, security] sources: [] updated: 2026-06-29 --- # Wayland Wayland is a communication protocol (plus a reference C library) that specifies how a display server talks to its clients, intended as the secure successor to the X Window System (X11). A display server speaking the protocol is called a *Wayland compositor*, because it merges the roles of display server, window manager, and compositing manager into a single process — eliminating X11's split between a separate X server, window manager, and compositor.[^1] It was started by Kristian Høgsberg as a free, community-driven project; the modern Linux desktop has been migrating to it for years, with GNOME and KDE Plasma defaulting to Wayland sessions and compositors like KWin, Mutter (GNOME Shell), Hyprland, Sway (wlroots), and COSMIC implementing the protocol.[^1] Unlike X11's client-server model from 1984 that routes rendering through multiple programs, Wayland has clients render their own buffers and hand them directly to the compositor, which reduces latency and eliminates screen tearing.[^1] The architectural shift matters most for security and automation.
